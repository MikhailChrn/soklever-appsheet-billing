## §1. Что произошло за #14 (12.05.2026)

### §1.1. Мини-патчи перед #08 (UX-правки)

**`students[birth_date]` Type=Text + подсказка.** Мобильный календарный пикер AppSheet требовал листать назад много месяцев для дат рождения 10+ лет — утомительно. Перешли на текстовый ввод с подсказкой `«дата рождения (дд.мм.гггг)»` в Display name. Валидация формата отклонена (поле справочное, §2.3 «суровая правда без шума»). По пути зафиксирована в SKILL фундаментальная особенность AppSheet: regex-функции (`MATCHES`/`ISMATCH`/`REGEXMATCH`) **не существуют** — это функции Google Sheets / Apps Script, не AppSheet. Подтверждено через Gemini и официальную документацию. Канон валидации даты-как-текста в AppSheet — `DATE(SUBSTITUTE([col], ".", "/"))`, зависит от Locale. В проекте не применяется.

**Удаление `expenses[recipient]`.** Владелец проанализировал реальные расходы (13 строк на момент решения), увидел избыточность поля «получатель»: где он критичен (педагоги, поставщики) — есть в `comment`; где не нужен (аренда, коммуналка) — пусто. Поле удалено в Sheets, AppSheet подхватил через Regenerate. **Сторона**: `SYSTEM.md §4.7` всё ещё упоминает «получателя» как часть сущности `expense` — не правил, потому что архитектурно поле было правильное, удалено по факту избыточности. Если в проде вернётся желание учитывать получателя — добавим обратно.

### §1.3. §4.5 — объединение плитки «Последнее закрытие» → «Последний скрипт»

**Возникло в живом диалоге.** Sonnet в первой редакции отчёта попытался «увильнуть» от §4.4.2 брифа (опциональной плитки в view «скрипты»), потому что бриф разрешал пропустить. Владелец вернулся к вопросу: «хочу видеть сообщение на дашборде». Стратег первоначально предложил отдельную плитку «Последний перевод на летний» по симметрии с «Последним закрытием» — владелец отклонил: «перевели — прочитали, закрыли — прочитали; одно должно заместить другое».

**Решение:** не плодить вторую плитку, а **обобщить существующую**. Колонки `settings.last_close_at`/`last_close_message` переименованы в `last_event_at`/`last_event_message`. Оба бота (`bot_close_period` и `bot_summer_apply`) пишут в общие поля. Кто отработал позже — того сообщение видно. Если появится третий скрипт — автоматически впишется. Заголовок плитки — **«Последний скрипт»** (отклонение Sonnet'а от брифа «Последнее действие», принято владельцем).

**Паттерн в SKILL:** Sonnet попутно обнаружил, что добавление нового Prominent action на `settings` автоматически прикрепило кнопку ко всем dash-slice'ам через Auto assign — кнопки расплодились по всему дашборду. Исправлено удалением action'а из Slice Actions каждого dash-slice'а. Зафиксировано в SKILL.

### §1.4. Мини-патч после отчёта — Initial value на `last_event_at`

После Regenerate Schema AppSheet автоматически проставил Initial value = NOW() на DateTime-колонку `last_event_at`. Sonnet это заметил и вынес в хвосты §11. Владелец сразу убрал. **Паттерн в SKILL:** для колонок, в которые пишут только боты, Initial value должна быть пустой, иначе при создании новой строки (если такое произойдёт) запишется время создания вместо времени последнего скриптового события.
