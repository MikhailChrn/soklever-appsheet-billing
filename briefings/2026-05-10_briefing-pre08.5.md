### §4.4. Запрет Edit для charges из закрытых периодов

**Контекст:** «твёрдая земля» (`SYSTEM.md` §2.2, Р-7) — закрытый период не правится задним числом. Это касается **всех** charges из закрытого периода, не только «Авто». Ручных charges в закрытом периоде по нормальному флоу не возникает — закрытие фиксирует то, что уже введено; корректировки задним числом делаются **новой** Ручной charge в текущем (открытом) периоде, при необходимости с минусом (Р-14 «суровая правда без шума»).

**Цель:** в карточке charges кнопка «редактировать» (системный Edit) видна **только** для charges текущего открытого периода. Для закрытых — скрыта, независимо от `charge_type`.

**Условие:** скриншот владельца от 11.05.2026 подтвердил — у системного Edit на `charges` свойство **Only if this condition is true** доступно (не read-only, в отличие от Target системного Add — см. SKILL Q.17). То есть здесь Show_If работает.

#### §4.4.1. Найти системный Edit для charges

| # | Шаг |
|---|-----|
| 4.4.1.1 | Behavior → Actions → `charges` → системный action `Edit` (System generated) |
| 4.4.1.2 | Раскрыть секцию **Behavior** — там поле **Only if this condition is true** |

#### §4.4.2. Применить Show_If

| # | Параметр | Значение |
|---|----------|----------|
| 4.4.2.1 | Only if this condition is true | `[period] = ANY(settings[current_period])` |

**Замечание по формуле:** `settings` — таблица из одной строки, прямое обращение `settings[current_period]` в AppSheet возвращает list (даже из одной строки). Сравнение `[period] = settings[current_period]` может не сработать как ожидается — список против скаляра. Корректный паттерн — обернуть в `ANY(...)`, чтобы получить скаляр. **Sonnet:** перед сохранением проверить через Test, что условие возвращает правильный TRUE/FALSE. Если паттерн `ANY(settings[current_period])` в проекте уже используется в других местах (например, в формуле плитки дашборда) — использовать тот же паттерн. Если используется иной (например, `INDEX(settings[current_period], 1)` или `MAX(settings[current_period])`) — взять его. **Сначала grep по `current_period` в Editor, потом писать формулу.**

#### §4.4.3. Тест

| # | Шаг | Ожидаемое |
|---|-----|-----------|
| 4.4.3.1 | Открыть карточку charge из **текущего открытого периода** (любой тип) | Кнопка «редактировать» видна |
| 4.4.3.2 | Открыть карточку charge из **любого закрытого периода** (Авто или Ручное) | Кнопка «редактировать» скрыта |
| 4.4.3.3 | Save в форме редактирования charge из открытого периода работает | — |

**В отчёт:** итоговая формула Show_If (точный синтаксис, с учётом проверки), результат тестов.

**Если в проекте `current_period` хранится не в `settings[current_period]`, а где-то ещё** (например, как VC `_current_period` на какой-то таблице — нужно проверить) — использовать тот источник. Сверить по SYSTEM.md §4.4 «Период» и формулам бота `bot_close_period`.

### §4.5. Фиксация источника `charges[charge_type]`

**Контекст:** после §4.4 «твёрдая земля» гарантируется через `period`, не через `charge_type`. Но `charge_type` остаётся семантическим маркером происхождения charge (отображается в карточке, используется в фильтрах, попадёт в отчётность). Поэтому источник должен ставиться **надёжно**, без зависимости от того, как пользователь заполнит форму.

**Цель:**
1. При создании charge через **форму** (человек) — `charge_type` автоматически = `"Ручное"`, пользователь не выбирает.
2. При создании charge **ботом** `bot_close_period` — `charge_type` явно прописан как `"Авто"` в шаге Add a new row to charges.

#### §4.5.1. Настройка колонки `charges[charge_type]`

| # | Шаг |
|---|-----|
| 4.5.1.1 | Data → Columns → `charges` → `charge_type` |
| 4.5.1.2 | **Initial value** = `"Ручное"` (с кавычками — Text) |
| 4.5.1.3 | **Editable?** = false **ИЛИ** убрать `charge_type` из Column order формы `charges_Form` (Show? в форме не сработает корректно — Show? влияет на видимость, не на редактируемость; правильнее Editable? = false, тогда колонка остаётся видна в Detail, но в форме недоступна) |
| 4.5.1.4 | Если Editable? = false ломает работу бота (бот тоже пишет в эту колонку) — тогда оставить Editable? = true, но **убрать** `charge_type` из Column order формы `charges_Form`. Решить по факту тестирования. |

**Sonnet:** сначала проверить, как сейчас работает форма создания charges руками (если такой путь вообще предусмотрен в UI — может, charges создаются **только** ботом, и тогда §4.5.1 формальная). См. контекст ниже.

**Контекст «как создаются ручные charges»:** проверить в UI — есть ли точка входа «+» на view начислений, или charges создаются только из карточки ученика через Related, или только ботом. Это меняет, какую `*_Form` править. **Перед изменениями — найти, через какие действия в UI пользователь создаёт Ручное начисление.** Зафиксировать в отчёте.

#### §4.5.2. Проверка бота `bot_close_period`

| # | Шаг |
|---|-----|
| 4.5.2.1 | Behavior → Bots → `bot_close_period` → найти шаг типа Action / Data, который создаёт charge (`Add a new row to charges` или похожий) |
| 4.5.2.2 | Открыть Set these columns — список присваиваний |
| 4.5.2.3 | Проверить, есть ли строка `charge_type = "Авто"` (или эквивалентная). **Если есть** — ок, ничего не трогать. **Если нет** — добавить явное присваивание `charge_type = "Авто"`. |

**Важно:** если бот сейчас полагается на Initial value колонки, то после §4.5.1.2 (Initial value = «Ручное») бот будет создавать «Ручное» от своего имени — катастрофа для отчётности. §4.5.2 закрывает этот риск.

**Порядок выполнения §4.5:** сначала §4.5.2 (убедиться, что бот пишет явно), потом §4.5.1 (изменить Initial value). Если перевернуть — между шагами окно, в котором бот может неправильно записать charges при срабатывании.

#### §4.5.3. Тест

| # | Шаг | Ожидаемое |
|---|-----|-----------|
| 4.5.3.1 | Создать новую charge через форму (путь, найденный в §4.5.1) | `charge_type` автоматически = «Ручное», в форме поле либо скрыто, либо недоступно для выбора |
| 4.5.3.2 | (Опционально, если есть тестовый период) Прогнать `bot_close_period` на тестовом периоде | Созданные charges имеют `charge_type = "Авто"` |

**В отчёт:** где создаются Ручные charges в UI (точка входа), что сделано с Initial value/Editable, что обнаружено в боте.
