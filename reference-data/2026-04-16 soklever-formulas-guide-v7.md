# Инструкция по формулам Google Sheets — СО Клевер (v7, финальная)

**Дата:** 16 апреля 2026
**Для файла:** soklever-google-ancestor-v5.xlsx
**Статус:** протестировано, устойчиво к непредсказуемости Google Sheets при импорте

---

## Ключевая идея v7

Все формулы теперь **устойчивы к типу данных period** (число или текст — не важно). Google Sheets при импорте xlsx непредсказуемо угадывает тип, и формулы v5–v6 ломались в половине случаев. В v7 этой проблемы нет.

**Приём:** в листы `payments` и `expenses` добавлены служебные колонки `period_num` с числовым представлением period. QUERY и FILTER теперь работают с числовыми колонками — совпадения гарантированы.

---

## Общие принципы

**Локаль:** все формулы под русскую локаль (разделитель `;`). Для английской — заменить `;` на `,`. Внутри кавычек QUERY запятые не трогать.

**ARRAYFORMULA** — для построчных вычислений на целых колонках (VLOOKUP, арифметика, конкатенация).

**MAP + LAMBDA** — для SUMIFS по парам значений (SUMIFS в ARRAYFORMULA работает неправильно).

**SUMPRODUCT** — для условных сумм с диапазонами периодов (SUMIFS со сравнением `">="&value` не работает для числовых данных).

**`*1` и `IFERROR(...*1; 0)`** — паттерн приведения значения к числу, устойчивый к обоим типам данных.

## ЧАСТЬ 1 — Вспомогательные колонки period_num

Это первым делом, чтобы дальше QUERY/FILTER работали правильно.

### Шаг 1.1 — payments!J2 (числовой period)

```
=ARRAYFORMULA(IF(D2:D=""; ""; D2:D*1))
```

Одна формула — колонка J заполнится числовым period автоматически.

### Шаг 1.2 — expenses!I2 (числовой period)

```
=ARRAYFORMULA(IF(C2:C=""; ""; C2:C*1))
```

### Шаг 1.3 — payments!I2 (класс ученика — как раньше)

```
=ARRAYFORMULA(IF(B2:B=""; ""; IFERROR(VLOOKUP(B2:B; students!A:E; 5; FALSE); "")))
```

## ЧАСТЬ 2 — Лист `debts`

Перед началом: если на листе что-то есть под заголовками — выделить `A2:I1000` → Delete.

Каждая формула ставится **один раз в строку 2**. Протягивать не нужно.

### Шаг 2.1 — A2 (student_id)

```
=ARRAYFORMULA(IF(charges!B2:B=""; ""; charges!B2:B))
```

### Шаг 2.2 — E2 (period)

```
=ARRAYFORMULA(IF(charges!B2:B=""; ""; charges!C2:C))
```

### Шаг 2.3 — F2 (charged)

```
=ARRAYFORMULA(IF(charges!B2:B=""; ""; charges!E2:E))
```

### Шаг 2.4 — B2 (student_name)

```
=ARRAYFORMULA(IF(A2:A=""; ""; VLOOKUP(A2:A; students!A:D; 3; FALSE) & " " & VLOOKUP(A2:A; students!A:D; 4; FALSE)))
```

### Шаг 2.5 — C2 (family_name)

```
=ARRAYFORMULA(IF(A2:A=""; ""; VLOOKUP(VLOOKUP(A2:A; students!A:B; 2; FALSE); families!A:B; 2; FALSE)))
```

### Шаг 2.6 — D2 (class_group)

```
=ARRAYFORMULA(IF(A2:A=""; ""; VLOOKUP(A2:A; students!A:E; 5; FALSE)))
```

### Шаг 2.7 — G2 (paid)

```
=MAP(A2:A; E2:E; LAMBDA(s; p; IF(s=""; ""; SUMIFS(payments!E:E; payments!B:B; s; payments!D:D; p))))
```

Используется MAP+LAMBDA вместо ARRAYFORMULA, потому что SUMIFS в ARRAYFORMULA берёт только первое значение пары и применяет ко всем строкам.

### Шаг 2.8 — H2 (opening_balance)

```
=ARRAYFORMULA(IF(A2:A=""; ""; IFERROR(-VLOOKUP(A2:A; opening_balances!B:D; 3; FALSE); 0)))
```

### Шаг 2.9 — I2 (debt)

```
=ARRAYFORMULA(IF(A2:A=""; ""; F2:F-G2:G+H2:H))
```

### Проверка `debts`

10 строк. Ключевой маркер: **Артём Петров за 202603: charged=18 000, paid=0, opening_balance=5 000, debt=23 000 ₽**.

У Алисы будут минусовые значения debt (`−1 000`, `−15 000`) — это корректно: 2 начисления + 1 платёж. В списке должников на дашборде они не появятся (там фильтр `debt > 0`).

## ЧАСТЬ 3 — Лист `dashboard`

Разметка в xlsx-v5:
- Строки 5–8: Текущий месяц
- Строки 10–13: С начала учебного года
- Строка 15: Расходы по категориям (до 9 строк результата)
- Строка 30: Поступления по классам (до N строк)
- Строка 45: **Должники — последний блок**, без ограничения сверху

Период `B3` уже содержит `202604` (число).

### Шаг 3.1 — B6 (поступления за месяц)

```
=SUMIFS(payments!E:E; payments!D:D; B3)
```

### Шаг 3.2 — B7 (расходы за месяц)

```
=SUMIFS(expenses!E:E; expenses!C:C; B3)
```

### Шаг 3.3 — B8 (свободные за месяц)

```
=B6-B7
```

### Шаг 3.4 — B11 (накопительные поступления)

```
=SUMPRODUCT((IFERROR(payments!D2:D1000*1; 0)>=(LEFT(B3&"";4)-1)*100+9)*(IFERROR(payments!D2:D1000*1; 0)<=B3*1)*payments!E2:E1000)
```

### Шаг 3.5 — B12 (накопительные расходы)

```
=SUMPRODUCT((IFERROR(expenses!C2:C1000*1; 0)>=(LEFT(B3&"";4)-1)*100+9)*(IFERROR(expenses!C2:C1000*1; 0)<=B3*1)*expenses!E2:E1000)
```

### Шаг 3.6 — B13 (накопительные свободные)

```
=B11-B12
```

### Шаг 3.7 — A16 (расходы по категориям)

Сначала удалить заглушку из A16. Затем:

```
=QUERY(expenses!A:I; "SELECT D, SUM(E) WHERE I = "&B3*1&" GROUP BY D ORDER BY SUM(E) DESC LABEL D 'Категория', SUM(E) 'Сумма'")
```

Использует колонку `period_num` (I) из expenses — числовое сравнение работает надёжно.

### Шаг 3.8 — A31 (поступления по классам)

Удалить заглушку из A31. Затем:

```
=QUERY(payments!A:J; "SELECT I, SUM(E) WHERE J = "&B3*1&" AND I IS NOT NULL GROUP BY I ORDER BY SUM(E) DESC LABEL I 'Класс', SUM(E) 'Поступления'")
```

Использует `class_group` (I) и `period_num` (J) из payments.

### Шаг 3.9 — A46 (должники)

Удалить заглушку из A46. Затем:

```
=FILTER(debts!B2:I1000; debts!I2:I1000>0; IFERROR(debts!E2:E1000*1;0)<B3*1)
```

`*1` приводит обе стороны сравнения к числу — работает независимо от типа period в debts.

## Если что-то не работает

1. **`#ERROR!`** сразу после вставки — проверь локаль (`;` vs `,`).
2. **Формула показывается текстом** — ячейка в текстовом формате. Формат → Число → Число, перевставить.
3. **QUERY возвращает только заголовки** — значит условие WHERE не нашло совпадений. Проверь через `=ISNUMBER(expenses!I2)` — должно быть TRUE. Если FALSE — формула в шаге 1.2 не применилась.
4. **FILTER даёт `#N/A`** — часто из-за несовпадения типов. Все `*1` в формуле должны быть на месте.
5. **FILTER даёт `#REF!` c сообщением «Массив не выведен»** — внизу листа мало места. Но в разметке v5 должник стоит последним, под ним пусто — такого быть не должно.
6. **Все значения paid одинаковые** — формула G2 старая. Должна быть `MAP+LAMBDA`.
7. **B11/B12 = 0** — формулы старые. Должны быть с `IFERROR(...*1; 0)`.
