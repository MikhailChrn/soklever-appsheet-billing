## §2. Сценарий переноса — финальный

В брифе был план Gemini (Share → Copy → переподключение Storage в каждой таблице). Финальный путь оказался чище:

1. **Share** приложения с A на B с правом «View/copy app».
2. На B: Shared with me → Copy → имя `soklever-root-billing v.1.1`. Тумблер `Make a copy of the data` оставлен включённым.
3. **Удалили автокопию** книги, которую AppSheet создал на Диске B (она получила имя `families` и содержала данные с аккаунта A — тестовые семьи).
4. Повторили **Copy app** — на этот раз **с выключенным** тумблером `Make a copy of the data`.
5. Открыли каждую из 13 таблиц в редакторе → Table settings (шестерёнка ⚙️) → Storage:
   - **Source Path** → выбрали наш эталон `2026-05-15 soklever-schema-v1.1` (Google Sheets на Диске B).
   - **Worksheet Name/Qualifier** → выбрали имя соответствующего листа.
   - Done.
6. **Save** в правом верхнем углу редактора (один раз для всех 13).

**Важно:** без шага «удалить автокопию + повторный Copy без данных» путь работает, но получается мусорная книга на Диске B, которую всё равно надо удалять. Чище — сразу копировать без данных.

**Также:** `tariffs` в копии скопировалась с режимом Updates + Adds + Deletes (как было в доноре). Поправили на **только Updates** (см. §3 — это перенесённый дефект донора).

## §3. Перенесённые дефекты донора (выявлены при раскатке)

### §3.1. Кнопка «перевести всех на летний» на view-фильтрах

Action `bot_summer_apply_trigger` отображается на view-фильтрах (`фильтр начислений`, `фильтр платежей`), где ему не место. Это auto-assign на dash-slice (паттерн зафиксирован в SKILL после #08).

**Уже есть в доноре v.1.0.** Владелец увидел при тестировании копии и подтвердил, что в доноре тоже виден — пропущено в #08.

**Решение:** будет исправлено в доноре v.1.0 в текущей сессии после отчёта.

### §3.2. `tariffs` — избыточные права Adds/Deletes

В доноре `tariffs` стоял Updates + Adds + Deletes. По логике справочника тарифов должно быть только Updates (изменение цены), без Adds/Deletes из приложения.

**Уже есть в доноре v.1.0.** Поправили в копии (Updates only). В доноре исправим в той же сессии.

Передача приложения между аккаунтами штатно работает через **Share + Copy на принимающей стороне** (Transfer ownership не используется). Apps Script и PDF-шаблоны у нас не используются — упрощает процедуру переноса.

### §8.2. Ошибка стратега — не сразу понял путь подключения

При раскатке стратег предлагал переключать Source Path вручную в каждой таблице (по совету Gemini). Владелец нашёл правильный путь сам: **удалить автокопию + Copy без данных** — приложение само подцепило листы эталона по именам. Это короче и чище.

Gemini дал общую раскладку (Share + Copy + Reconnect), которая в целом верна, но конкретный UI-путь не угадал. По одному узкому вопросу (где Storage в новом интерфейсе) Gemini ответил мимо контекста — пришлось переспрашивать. Принцип: Gemini — для архитектурного «как» в общих чертах; конкретику в UI — проверяем руками.
