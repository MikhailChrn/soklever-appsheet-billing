## §3. Задача §4.1 — Заголовки при редактировании (6 сущностей)

### Архитектурное решение

Исходный план брифинга (два form view + `LINKTOROW`) не реализован из-за ограничения AppSheet (май 2026): невозможно надёжно разделить форму Add и форму Edit через два form view на одну таблицу. Подробности — в §7 (смежные хвосты / блокеры).

**Принятый компромисс (согласован со стратегом):** универсальный заголовок `«Создать/редактировать <сущность>»` через виртуальную колонку `_form_header_<table>` (тип Show, категория Section_Header, Display name = заголовок).

### Попытки разделения Add/Edit (задокументировано для скилла)

В ходе сессии были опробованы следующие подходы — все не дали результата:
1. Создание `z_tariffs_Form_Edit` с Position = ref — AppSheet всё равно использовал Edit-форму для «+».
2. Переименование в `z_*` — не помогло.
3. `App: open a form to add a new row` — не даёт выбрать целевую view в текущей версии редактора.
4. Event Actions в deck view — поля Add нет.

Все промежуточные артефакты (`z_tariffs_Form_Edit`, `tariffs__edit`, `_form_header_tariffs_edit`) удалены.

## §9. Что добавить в appsheet-setup-SKILL.md

Подробная выжимка — в файле `skill-additions-pre08-3.md`. Краткое резюме:

**1. Новый Q-блок: Form header через Section_Header VC**

В текущей версии AppSheet невозможно надёжно направить Add и Edit на разные form view одной таблицы. Все пути опробованы и задокументированы. Компромисс — универсальный заголовок `«Создать/редактировать <X>»`.
