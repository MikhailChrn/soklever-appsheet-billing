## §3. Задача §4.1 — Удаление `tariff_type`

**Выполнена полностью.**

**Старая формула фильтра в `bot_close_period` → `step_no_create_charges` → Referenced rows:**
```
SELECT(student_tariffs[assignment_id],
  AND(
    [tariff_id].[tariff_type] = "Регулярный",
    [student_id].[status] = "Активен"
  )
)
```

**Новая формула:**
```
SELECT(student_tariffs[assignment_id],
  [student_id].[status] = "Активен"
)
```

**Дополнительная ссылка на `tariff_type`** обнаружена при Regenerate Schema — VC `settings[current_period]`:

Старая:
```
INDEX(SORT(SELECT(charges[period], [tariff_id].[tariff_type] = "Регулярный"), TRUE), 1)
```
Новая:
```
INDEX(SORT(SELECT(charges[period], TRUE), TRUE), 1)
```
Исправлена в ходе сессии (ошибка всплыла как Errors & Warnings при Regenerate Schema).

**Test формулы бота:** вернул список A003, A004, A005, 2b8fc2d3 — 4 строки, ошибок нет.
